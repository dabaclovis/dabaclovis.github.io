Docker is a Containerization tool

Virtualization: Fixed hardware allocation

Containerization: No Fixed hardware allocation

Process isolation

Docker implements a high-level api to provide lightweight containers that run processes in isolation.

Docker containers eliminate the need for a separate guess operating system for every new virtual machine.

Docker containers enables rapid deployment with minimal run-time requirements.

Docker containers ensures better management and simplified portability

Docker containers help developers and operational team in rapid deployment of application

key words to note:

  • Docker images: Combination of binaries which are necessary for software application
  • Docker Container: When an image is executed and it's in a running state, it's called container
  • Docker Host: Machine on which docker is installed
  • Docker Client: Terminal used to run docker commands

Working with images:

  1. Command to download a docker image.
    • Docker pull image_name
    • Example: docker pull nginx
  2. See list of docker images:
    • Docker image ls
    • Docker images
      •
  3. Delete an image from a docker host.
    • docker rmi image_name/image_id
  4. Push docker image to dockerhub.
    • Example
    • Docker push image_name
  5. Create docker image from dockerfile:
    • Example
    • docker build -t new_image_name
  6. Search for a docker image:
    • docker search image_name
  7. To delete all images:
    • docker system prune -a
